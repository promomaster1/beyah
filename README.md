# نظام EPOS - نظام تتبع مؤشرات الأداء
## جمعية البيئة بحائل

نظام داخلي بسيط لتتبع مؤشرات الأداء الخاصة بجمعية البيئة بحائل.

## 🎯 المميزات

### المحاور الخمسة الثابتة:
1. البرامج البيئية التطبيقية ذات الأثر
2. البناء المؤسسي والحوكمة التشغيلية
3. الاستدامة المالية وبناء الشراكات
4. المشاركة المجتمعية وقياس الأثر
5. مؤشرات مستوى الكيان

### الوظائف الأساسية:
- ✅ لوحة معلومات تفاعلية مع نظام ألوان (أخضر/أصفر/أحمر)
- ✅ إدخال بيانات ربع سنوية للمؤشرات
- ✅ إدارة الأهداف السنوية
- ✅ إمكانية إضافة مؤشرات جديدة
- ✅ تقارير سنوية قابلة للطباعة (PDF)
- ✅ رسوم بيانية توضيحية
- ✅ 3 أدوار مستخدمين (مدير، مدخل بيانات، مشاهد)

## 🔧 التقنيات المستخدمة

### Backend:
- FastAPI (Python)
- MongoDB
- JWT Authentication
- Motor (Async MongoDB driver)

### Frontend:
- React 19
- TailwindCSS
- Recharts (للرسوم البيانية)
- jsPDF (لتوليد PDF)
- Radix UI Components
- Sonner (للإشعارات)

## 📦 التثبيت

### المتطلبات:
- Python 3.11+
- Node.js 18+
- MongoDB

### خطوات التشغيل:

1. **Backend Setup:**
```bash
cd /app/backend
pip install -r requirements.txt
python init_data.py  # إدخال البيانات الأولية
```

2. **Frontend Setup:**
```bash
cd /app/frontend
yarn install
```

3. **تشغيل الخدمات:**
```bash
sudo supervisorctl restart all
```

## 📊 هيكل قاعدة البيانات

### Collections:
- **users** - بيانات المستخدمين
- **axes** - المحاور الخمسة
- **indicators** - المؤشرات (50 مؤشر)
- **targets** - الأهداف السنوية
- **values** - القيم الربع سنوية

## 🎨 الواجهة

### الصفحات الرئيسية:
1. **لوحة المعلومات** (`/dashboard`)
   - عرض الأداء الإجمالي
   - بطاقات المحاور الخمسة بألوان حسب الأداء
   - إمكانية اختيار السنة

2. **تفاصيل المحور** (`/axis/:id`)
   - عرض جميع مؤشرات المحور
   - رسوم بيانية ربع سنوية
   - مقارنة الهدف بالإنجاز

3. **إدخال البيانات** (`/data-entry`)
   - نموذج بسيط لإدخال القيم
   - اختيار السنة والربع والمحور والمؤشر
   - التحقق من البيانات

4. **إدارة الأهداف** (`/targets`) - Admin فقط
   - تحديد الأهداف السنوية لكل مؤشر

5. **إدارة المؤشرات** (`/indicators`) - Admin فقط
   - إضافة مؤشرات جديدة
   - عرض جميع المؤشرات حسب المحور

6. **التقارير** (`/reports`)
   - عرض التقرير السنوي
   - توليد PDF قابل للطباعة

7. **إدارة المستخدمين** (`/users`) - Admin فقط
   - إضافة مستخدمين جدد
   - تحديد الأدوار

## 📄 API Endpoints

### Authentication:
- `POST /api/auth/login` - تسجيل الدخول
- `GET /api/auth/me` - بيانات المستخدم الحالي

### Axes:
- `GET /api/axes` - قائمة المحاور
- `GET /api/axes/{id}` - تفاصيل محور

### Indicators:
- `GET /api/indicators` - قائمة المؤشرات
- `POST /api/indicators` - إضافة مؤشر (Admin)
- `PUT /api/indicators/{id}` - تعديل مؤشر (Admin)

### Targets:
- `GET /api/targets?year={year}` - الأهداف لسنة محددة
- `POST /api/targets` - إضافة/تعديل هدف (Admin)

### Values:
- `GET /api/values` - القيم مع إمكانية التصفية
- `POST /api/values` - إضافة قيمة
- `PUT /api/values/{id}` - تعديل قيمة

### Dashboard:
- `GET /api/dashboard?year={year}` - بيانات اللوحة الرئيسية
- `GET /api/dashboard/axis/{id}?year={year}` - تفاصيل محور

### Reports:
- `GET /api/reports/annual?year={year}` - التقرير السنوي

### Users:
- `GET /api/users` - قائمة المستخدمين (Admin)
- `POST /api/users` - إضافة مستخدم (Admin)
